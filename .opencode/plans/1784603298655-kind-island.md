# Extracción Completa Y Evidencia De Auditoría

## Objetivo

Ampliar el scraper autenticado para capturar evidencia visual local, auditar la
configuración completa de Open to Work y validar que Skills y Certifications
estén realmente completos. El informe seguirá evitando conclusiones cuando la
evidencia no sea verificable.

## Decisiones Cerradas

- Capturar foto de perfil y banner en cada `npm run scrape` como PNG locales;
  el agente pide consentimiento separado **en cada auditoría** antes de abrir o
  enviar las imágenes al proveedor de IA.
- Extraer configuración completa de Open to Work, pero mostrar en `report.md`
  solo un resumen privado por defecto: estado, visibilidad y alineación con el
  objetivo. Títulos y ubicaciones completos se muestran solo si el usuario lo
  solicita.
- Si Skills o Certifications siguen incompletos tras reintentos, la extracción
  termina con aviso y la auditoría continúa, pero excluye esas secciones de
  scoring y recomendaciones formales.
- El scraper no modifica LinkedIn, no descarga URLs de imágenes ni lee/imprime
  cookies o estado de autenticación.

## Implementación

### 1. Modelo privado y escritura atómica

Modificar `src/scraping/scrape-profile.mjs` para generar, junto al texto:

- `data/profile/profile.json` con modo `0600` y reemplazo atómico.
- `data/profile/visual/profile-header.png`, `profile-photo.png` y
  `profile-banner.png`, cada uno con modo `0600` y reemplazo atómico.

Mantener `profile.txt` para compatibilidad. Incluir en JSON:

- `schemaVersion`, fecha y URL canónica solo como procedencia local.
- Estado por sección: `complete`, `partial`, `unavailable`.
- Conteos declarados/extraídos, reintentos y advertencias.
- Estado/rutas de cada evidencia visual: `captured`, `notPresent`,
  `unavailable`.
- Estado y datos estructurados de Open to Work.

No repetir URL completa, rutas locales ni datos privados innecesarios en el
informe de IA.

### 2. Extracción semántica por sección

Extraer de `src/scraping/scrape-profile.mjs` y añadir módulos puros bajo
`src/scraping/` para:

- Definir expectativas por sección (heading, contenedor/lista, registros y
  conteos).
- Expandir controles en enlaces y botones, en inglés/español, reconsultando y
  desplazándose hasta que el número de registros se estabilice.
- Validar contenido real, no el umbral actual de 80 caracteres.
- Deduplicar registros visibles conservando orden.
- Reintentar solo Skills y Certifications, con navegación nueva y backoff
  acotado (tres intentos totales).

Skills requiere encabezado, registros reales y carga agotada; Certifications
requiere encabezado y entradas de credencial reales. Si LinkedIn declara un
total, reconciliarlo con el total extraído. Una página con UI/footer sin
registros será `unavailable`, no `complete`.

Experience, Education y Recommendations deben distinguir una sección vacía
válida de una página incompleta. Los fallos de autenticación, perfil propio o
contenido principal requerido siguen siendo fatales y preservan los archivos
anteriores.

### 3. Evidencia visual acotada

Tras validar el perfil propio, abrir su página principal y capturar únicamente:

- El elemento de foto dentro del encabezado/intro del perfil.
- El elemento de portada dentro del encabezado, incluyendo fondos CSS mediante
  screenshot del locator.
- Un encabezado acotado como evidencia de composición, solo si puede aislarse
  sin capturar actividad, conexiones, mensajes o recomendaciones.

Usar locators defensivos, scoped al encabezado, y tratar ausencia/múltiples
coincidencias como advertencia no fatal. No usar screenshots de `body`, `main`
completo o página completa.

### 4. Open To Work de solo lectura

Añadir una extracción separada en `src/scraping/scrape-profile.mjs` o módulo
dedicado que, usando la sesión autenticada:

1. Navegue desde Jobs hacia Preferences y Open to Work mediante roles/nombres
   accesibles con variantes inglés/español.
2. Abra el panel/modal de configuración solo para leerlo.
3. Extraiga títulos, ubicaciones, workplace types, employment types y
   visibilidad, sin pulsar Save ni cambiar valores.
4. Verifique authwall/checkpoint después de cada transición y cierre/navegue
   fuera sin enviar cambios.

Persistir estados sin inferencias:

- `enabled`, `disabled`, `notConfigured`, `unavailable`.
- Visibilidad: `recruitersOnly`, `allMembers`, `onlyYou`, `unknown`.

La ausencia de una tarjeta pública nunca equivale a que Open to Work esté
inactivo. Si no se puede verificar el panel, marcar `unavailable`; si está
inactivo o no configurado de forma positiva, informar al usuario y la auditoría
debe recomendar activarlo solo cuando su objetivo laboral lo justifique.

Agregar al texto una breve sección rotulada de Open to Work con estado y
advertencias. El detalle estructurado queda en JSON para el agente.

### 5. Skill, consentimiento e informe

Cambios:

- Añadir tabla de completitud de extracción al inicio del informe.
- Evaluar foto/banner solo si los PNG existen y hay consentimiento visual;

- Cuando Skills/Certifications sean `partial`/`unavailable`, excluir sus filas

  destacar la limitación cerca del alcance del informe.
- Añadir sección Open to Work con estado, visibilidad y alineación con objetivo;
  sugerir activarlo solo si se confirma inactivo/no configurado y el objetivo es

### 7. Pruebas y CI

Añadir fixtures sanitizados y tests Node sin LinkedIn ni `.auth/`:

- Skills/Certifications completos, vacíos válidos, shell-only, lazy-loaded,
  expandibles por enlaces, duplicados y conteos declarados discordantes.
- Reintentos agotados que generan scrape válido con estado `unavailable` y
  warning, no fallo total.
- Captura visual presente, ausente y no disponible; validación de rutas, modos
  `0600` y escritura temporal/atómica.
- Open to Work enabled/disabled/notConfigured/unavailable, visibilidades y
  campos en inglés/español.
- Transiciones authwall y página inesperada permanecen fatales.
- Validar skill/report template para los nuevos campos de completitud,
  consentimiento visual y Open to Work.

Ejecutar `npm run typecheck`, `npm test`, `npm run doctor`, render de banner y
una prueba manual autenticada de `npm run scrape`. En la prueba manual, validar
solo estados, conteos, paths y permisos; no imprimir perfil, configuración o
imágenes sensibles.
